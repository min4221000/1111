▶ BEFORE
pattern[turn % len(pattern)]
▶ AFTER
pattern[(turn - 1) % len(pattern)]

▶ BEFORE (버그)
_monster_turn_logic(...)

return False

▶ AFTER (수정)
prev_index = player.active_index
_monster_turn_logic(...)
if not _check_player_alive(player):
return False
if player.active_index != prev_index:
player.draw_cards(player.active_char, 4)

Design Changes (v6.2)
D17
DESIGN
_check_synthesis
재귀 → while 루프
player.py
연쇄 합성 처리를 위해 자기 자신을 재호출하던 재귀 구조를 while True / break 루프로 변경. 동작은 동일하나 코드 흐름이 선형적으로 읽힘. next(generator) 패턴도 일반 for 루프로 교체.
▶ BEFORE
def _check_synthesis(self):
...
self._check_synthesis() # 재귀
return
▶ AFTER
def _check_synthesis(self):
while True:
found = False
...
if not found: break
재귀 제거next(generator) → for 루프
D18
DESIGN
Counter → 일반 딕셔너리
player.py
show_detailed_status()의 카드 중복 집계에 collections.Counter를 사용했으나 기본 딕셔너리와 .get()으로 대체. Counter import 제거. 동작 동일.
▶ BEFORE
from collections import Counter
card_counts = Counter(c.get('deck', []))
▶ AFTER
card_counts = {}
for name in c.get('deck', []):
card_counts[name] = card_counts.get(name, 0) + 1
Counter import 제거기본 dict로 대체
