    def get_monster(name, floor):
        scale = 1 + (floor * 0.1)
        base["hp"]  = int(base["base_hp"]  * scale)   # HP: 층당 10% 증가

  ▸ deepcopy를 쓰는 이유
    MONSTERS는 원본 템플릿. deepcopy 없이 수정하면

    기존엔 SPD도 층당 5% 증가해 10층 이후 거의 모든 몬스터가
    플레이어보다 빠른 밸런스 문제가 있었음.
    보스와 동일하게 base_spd 고정값 사용으로 변경.

  [ 핵심 로직 3 ]  공용 카드는 이벤트로만 획득

    각각 특정 이벤트(약초상/수도승/비밀 통로/마력의 샘)에서만 획득.

  [ 핵심 로직 5 ]  상태이상 처리 분리

    def process_turn_statuses(entity):
        if status == 'stun':
            continue          # stun은 행동 시 소비 — 여기서 처리 안 함
        if status == 'burn':
            entity['hp'] -= 5
        elif status == 'poison':
            entity['hp'] -= turns   # 스택 수 = 데미지

  ▸ stun을 여기서 처리하지 않는 이유
    burn / poison  →  "턴 종료 시" 발동
    stun           →  "행동 시" 소비  (보스 면역 등 분기 포함)
    섞으면 타이밍 버그 발생.

  ▸ poison 데미지 = 현재 스택 수인 이유
    스택이 쌓일수록 초반이 강하고 시간이 지나면 약해지는
    독 특유의 감쇠 패턴 재현.

  ① Data-driven Design (데이터 주도 설계)
     카드/몬스터/유물을 딕셔너리 데이터로 정의.
     로직 수정 없이 콘텐츠만 추가/수정 가능.

  ② Single Responsibility (단일 책임)
     각 함수/파일이 하나의 역할만 담당.
     side_effects 패턴으로 책임 경계를 명확히 유지.

  ③ Graceful Degradation (점진적 저하)
     AI 기능처럼 선택적 의존성은 try/except + fallback으로
     기능 일부가 없어도 전체가 동작하도록 설계.

  ④ 단방향 의존성
     main → world → combat → player/assets 순서.
     하위 모듈이 상위를 import 하지 않아 순환 참조 없음.
